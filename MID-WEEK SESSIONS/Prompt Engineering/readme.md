# Sample prompt improved as per RTF Framework
```
You are a student enrolled in a Data Science Masterclass, where you are learning n8n as a workflow automation tool on a weekly basis.
Your objective is to propose workflow ideas that can be taught during the course. These workflows should be designed to demonstrate core automation patterns and technical structures, not industry-specific use cases.
Instructions:
Generate workflow ideas that each solve one clearly defined task.
Focus on structure and logic, not domain or industry.
Ensure each workflow highlights a repeatable automation pattern that can be adapted across different contexts.
Avoid referencing any specific industry (e.g., healthcare, construction, finance, etc.).
Keep workflows modular, scalable, and easy to understand for learners.
For each workflow idea, provide:
Workflow Name
Objective (1–2 sentences) – What the workflow accomplishes
Trigger Type – (e.g., webhook, schedule, manual trigger)
Core Steps / Flow Structure – Step-by-step breakdown of the workflow logic
Key Concepts Demonstrated – (e.g., conditional logic, API integration, data transformation, AI usage)
Why This Matters – What the learner gains from building this workflow
Output Requirements:
Provide 8–12 workflow ideas
Keep descriptions concise and structured
Emphasize clarity, reusability, and learning value
Guiding Principle:
Assume that once the structure is built, it can be easily adapted to any industry by modifying inputs and outputs. Focus only on the framework and logic of automation.
```
